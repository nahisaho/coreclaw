# CoreClaw 画面一覧・画面要件定義書

本書は、CoreClaw の現行 Web UI をもとに、主要画面とモーダルの構成、および各画面に求められる要件を整理したものである。

## 1. 画面構成概要

CoreClaw は単一ページアプリケーションとして構成され、以下の 2 ペインを基本とする。

- 左ペイン: サイドバー
- 右ペイン: メイン作業領域

右ペイン上で必要に応じて複数のモーダル画面を重ねて利用する。

|---|---|---|---|

| SCR-02 | 新規実験モーダル | モーダル | 実験の新規作成 |
| SCR-03 | 実験編集モーダル | モーダル | 実験名、説明、スキル、同期先、MCP の更新 |

| SCR-05 | 設定モーダル | モーダル | 全体設定の管理 |

## 3. 画面要件

### SCR-01 メイン画面

#### 目的

- 実験を選択し、会話、同期、ファイル操作、タスク監視を行う。

#### 表示要素

- サイドバー開閉ボタン
- アプリ名表示
- 新規実験ボタン
- 実験一覧
- 実験検索導線
- 設定ボタン
- サーバー停止ボタン
- 認証状態表示
- チャットヘッダ
- GitHub Push/Pull ボタン
- メッセージ一覧
- 入力欄
- 音声入力ボタン
- 送信ボタン
- 実行中ステータスパネル

#### 機能要件

- 実験一覧から現在対象の実験を切り替えられること。
- 実験のピン留め状態と状態種別を視覚的に区別できること。
- メッセージは Markdown ベースで閲覧できること。
- 実行中タスクの途中経過と経過時間を表示できること。
- GitHub への Push/Pull 操作を実行できること。
- 音声入力開始、停止、文字起こし反映ができること。
- 送信直後からエージェント応答完了までストリーミングを維持できること。

### SCR-02 新規実験モーダル

#### 入力項目

- 実験名
- 説明
- Skills
- Sync Repository
- MCP Servers

#### 機能要件

- 必要項目入力後に実験を作成できること。
- スキル選択 UI を呼び出せること。
- MCP サーバーの適用対象を選択できること。

### SCR-03 実験編集モーダル

#### 入力項目

- 実験名
- 説明
- Skills
- Sync Repository
- MCP Servers

#### 機能要件

- 現在設定値を初期表示できること。
- 実験属性を更新できること。
- スキルや同期先を変更した結果が実験に反映されること。

### SCR-04 実験削除確認モーダル

#### 機能要件

- 削除対象実験を利用者が認識できること。
- 明示的な確認操作があるまで削除を実行しないこと。

### SCR-05 設定モーダル

#### タブ構成

- General
- Authentication
- STT Provider
- Skills
- GitHub
- MCP Servers
- Updates

#### 共通要件

- タブ切り替えにより各設定領域を切り替えられること。
- 保存後に設定が永続化されること。
- 取得した秘密情報はマスク表示されること。

#### General タブ要件

- テーマまたは表示モードを切り替えられること。
- 出力言語を選択できること。

#### Authentication タブ要件

- GitHub Token を入力できること。
- Copilot モデルを選択できること。
- GitHub MCP Tools の範囲を設定できること。
- Copilot 認証状態を表示し、再確認できること。

#### STT Provider タブ要件

- OpenAI、Azure OpenAI、Browser を切り替えられること。
- 必要に応じて API Key、Endpoint を設定できること。

#### Skills タブ要件

- ローカルスキル一覧を表示できること。
- スキル新規作成、更新、削除、スキャンができること。
- Marketplace セクションへ遷移せず同画面内で状態確認できること。

#### GitHub タブ要件

- GitHub ユーザー名や Marketplace 用リポジトリ URL を設定できること。

#### MCP Servers タブ要件

- MCP サーバー一覧の追加、編集、削除ができること。
- プリセットから ToolUniverse や Deep Research を追加できること。
- command、args、env などの構成を入力できること。

#### Updates タブ要件

- 現在バージョンを表示できること。
- 更新確認と更新実行を行えること。

### SCR-06 成果物一覧モーダル

#### 機能要件

- 実験成果物をツリー表示できること。
- ディレクトリとファイルを区別表示できること。
- 選択した成果物をダウンロードまたはビューア表示へ遷移できること。

### SCR-07 成果物ビューアモーダル

#### 機能要件

- Markdown、Mermaid、画像、PDF、JSON、コードなどを表示できること。
- Raw 表示切り替えができること。
- 内容コピーができること。

### SCR-08 Marketplace 情報モーダル

#### 機能要件

- スキルグループ名、ソース、説明、バージョン等を表示できること。
- インポート判断に必要な情報を提示できること。

### SCR-09 メッセージ検索モーダル

#### 機能要件

- 実験内メッセージをキーワードで検索できること。
- 検索結果から対象会話位置を特定できること。

### SCR-10 活動ログモーダル

#### 機能要件

- タスク、ツール、コマンド、ファイル、モデル、システムのカテゴリで絞り込みできること。
- タスク単位で履歴を絞り込めること。
- 時系列でイベント一覧を確認できること。

### SCR-11 スキル選択ポップアップ

#### 機能要件

- スキルを検索・フィルタできること。
- 実験作成・編集の両文脈で同じ選択機構を利用できること。

## 4. 画面遷移の要約

- メイン画面から新規実験、編集、削除、設定、成果物、検索、活動ログへ遷移する。
- 新規実験・編集モーダルからスキル選択ポップアップを開く。
- 成果物一覧から成果物ビューアへ遷移する。
- 設定モーダル内で Skills タブから Marketplace 情報モーダルを開く。

## 5. UI 共通要件

- ダークテーマとライトテーマの双方に対応すること。
- モーダルは明示操作または背景クリックで閉じられること。
- 実行中の重要状態は視認しやすい色やバッジで提示すること。
- 長文コンテンツや履歴一覧はスクロール可能であること。
