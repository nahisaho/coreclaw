# CoreClaw 機能要件定義書

本書は、CoreClaw の現行コードベースをもとに、利用者視点で求められる機能要件を整理したものである。実装済みコードを根拠としているため、新規構想ではなく、現行版が提供している振る舞いの明文化を目的とする。

## 1. 文書の目的

- CoreClaw が提供する主要機能を、画面、API、実行基盤の観点で整理する。
- 今後の改修、テスト設計、利用者向け説明資料の基礎となる要求を定義する。
- 実装済み機能と、今後拡張可能な周辺機能を切り分ける。

## 2. 対象範囲

本書の対象範囲は以下とする。

- Web UI による実験管理、チャット、設定、成果物閲覧
- Node.js サーバーによる API、WebSocket、設定永続化、実験実行制御
- Docker コンテナ上で動作する GitHub Copilot CLI ベースのエージェント実行
- Agent Skills、MCP サーバー、GitHub 連携、音声入力、セルフアップデート

本書では、コード中に基盤として存在するメッセージングチャネル連携や定期実行機構も、システム機能として扱う。

## 3. 想定利用者

- 一般利用者: Web UI から対話、ファイルアップロード、成果物確認を行う利用者
- 管理者: 設定、スキル導入、MCP 追加、更新、停止を行う利用者
- 拡張開発者: Skills、MCP、GitHub 連携、外部チャネルを拡張する開発者

## 4. システム概要

CoreClaw は、Web フロントエンド、Node.js ベースの制御サーバー、Docker コンテナで分離実行されるエージェント処理から成る AI エージェント基盤である。ユーザーは実験単位で対話を管理し、各実験に対してスキル、MCP サーバー、同期リポジトリなどを設定できる。エージェントは GitHub Copilot CLI を使用してコンテナ内で実行され、進行状況や出力はリアルタイムに UI へ反映される。

## 5. 機能要件

|---|---|---|

| FR-05 | 実験状態管理 | システムは、実験状態として active、archived、completed を保持し、表示に反映できなければならない。 |

## 6. データ永続化要件

- 実験、メッセージ、タスク履歴、活動ログは SQLite ベースで永続化されること。
- 設定、スキャンホワイトリスト、各種ランタイムファイルは data 配下へ保存されること。
- 実験成果物は実験ごとのワークスペース配下へ保存されること。
- セッション ID、登録グループ、スケジュールタスクは再起動後も復元可能であること。

## 7. 画面要件の要約

- 2 ペイン構成の SPA であること。
- 左ペインに実験一覧、検索、設定、認証状態、停止導線を持つこと。
- 右ペインにチャット、進行中タスク、成果物、活動ログ、同期操作を持つこと。
- 設定モーダルから認証、STT、スキル、GitHub、MCP、更新を一元管理できること。

## 8. 本書で対象外とする事項

- 将来追加予定だが現行コードに存在しない新規業務要件
- Azure など特定クラウド固有の運用設計
- 詳細な非機能要件、性能要件、運用要件、テスト要件

## 9. 補足

本書は現行ソースコードの実装事実を整理したものである。そのため、今後の仕様変更時には API、UI、コンテナ実行フロー、設定項目の差分に応じて更新することを前提とする。
